OPPONENT INTELLIGENCE & PERSISTENCE SYSTEM

OBJECTIVE

Implement a persistent scouting database that continuously builds intelligence across multiple games. Every analyzed snap should contribute to long-term opponent profiles, user profiles, and learning systems.

This is not simply “saving data.” It is the foundation for the Madden Intelligence Engine.

⸻

MULTI-LAYER PERSISTENCE

Implement three persistence layers.

Layer 1 — Session Persistence

Purpose:
Prevent data loss during active analysis.

Persist:

* Current upload
* Current analysis progress
* Current snap index
* User corrections
* Pending reports
* Analysis settings

Refreshing the page should never lose an active session.

⸻

Layer 2 — Local Persistence

Store all completed analyses locally for fast access.

Persist:

* Snap logs
* Reports
* AI findings
* User corrections
* Confidence scores
* Opponent profiles
* User profiles

Previously analyzed games should load instantly without rebuilding everything.

⸻

Layer 3 — Cloud Persistence

When authentication is available, synchronize all scouting data.

Cloud storage becomes the permanent source of truth while local storage acts as a high-speed cache.

⸻
